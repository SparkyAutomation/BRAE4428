What is a PID Controller?

The Control Loop

A PID controller continuously calculates an error — the difference between a desired setpoint and the measured process value — then applies a correction based on three terms:

u(t) = Kp e(t)  +  Ki ∫e(t)dt  +  Kd de/dt

🔵 Proportional — Kp

Reacts to the current error. Higher Kp = faster response but risks overshoot and oscillation.

  • ↑ Kp → faster rise, more overshoot
  • ↓ Kp → slower, more stable
  • Alone: leaves a steady-state error

🟢 Integral — Ki

Reacts to accumulated past error. Eliminates steady-state error but can cause windup and overshoot.

  • ↑ Ki → eliminates offset faster
  • Too high → oscillation / instability
  • Fixes: steady-state error

🔴 Derivative — Kd

Reacts to the rate of change of error. Dampens oscillations but amplifies noise.

  • ↑ Kd → less overshoot, more damping
  • Too high → noisy, unstable
  • Fixes: overshoot / oscillation

Manual Tuning Method

1

Zero all gains

Set Kp=0, Ki=0, Kd=0. Start from a clean slate.

2

Increase Kp

Raise Kp until the output oscillates with constant amplitude. Note this as Kp,osc, then reduce to ~50%.

3

Add Ki

Increase Ki slowly until steady-state error is eliminated without excessive overshoot.

4

Add Kd

Increase Kd to reduce overshoot and improve settling time. Stop before noise becomes an issue.

5

Fine-tune

Iterate small adjustments. Priority: stability → no steady-state error → fast response.

💡 Rule of thumb: Oscillating? Reduce Kp or Kd. Too slow? Increase Kp. Steady-state error? Increase Ki.

Ziegler-Nichols Method

A systematic empirical method developed in 1942. Two variants:

Closed-Loop Oscillation Method

  1. Set Ki=0 and Kd=0 (P-only control)
  2. Increase Kp until output oscillates at constant amplitude → Ultimate Gain Ku
  3. Measure the oscillation period Tu
